# Terminal of Goblins — Spec

## Concept
Inspired by "Infinite Backrooms" by Andy Ayrey. Two AI goblin characters have infinite, automatically generated conversations with each other. No human intervention. The conversations use a CLI/terminal metaphor — the goblins are exploring a dark cave system through a command line.

## Tech Stack
- **Framework:** Next.js 15 (App Router)
- **Language:** TypeScript
- **AI:** OpenAI GPT-5.5 (two instances talking to each other)
- **Database:** PostgreSQL (Railway)
- **Real-time:** Server-Sent Events (SSE)
- **Styling:** Tailwind CSS
- **Deploy:** Railway

## Architecture

### Database Schema
```sql
-- Conversations (each is a session between two goblins)
CREATE TABLE conversations (
  id SERIAL PRIMARY KEY,
  title TEXT,
  goblin_a_name TEXT NOT NULL,
  goblin_b_name TEXT NOT NULL,
  status TEXT DEFAULT 'active', -- active, completed, error
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Messages within conversations
CREATE TABLE messages (
  id SERIAL PRIMARY KEY,
  conversation_id INTEGER REFERENCES conversations(id),
  role TEXT NOT NULL, -- 'goblin_a' or 'goblin_b'
  content TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Goblin System Prompts
Each goblin gets a unique personality. They interact through a simulated cave CLI.

**Goblin A (Grix):** A cunning, scheming goblin who speaks in broken English with goblin slang. Obsessed with finding treasure and traps. Uses the terminal to navigate cave systems.

**Goblin B (Murk):** A paranoid, philosophical goblin who questions reality. Sees patterns in everything. Uses the terminal to decode ancient goblin runes.

The conversation format simulates a shared terminal where both goblins type commands and see each other's output. They use CLI-style commands like:
- `dig <direction>` - dig tunnels
- `sniff` - smell for treasure/danger
- `cast <spell>` - goblin magic
- `whisper <message>` - secret communication
- `examine <object>` - look at things
- Custom made-up goblin commands

### API Routes

1. **GET /api/conversations** - List all conversations (paginated)
2. **GET /api/conversations/[id]** - Get a specific conversation with messages
3. **POST /api/conversations/generate** - Trigger generation of a new conversation
4. **GET /api/conversations/[id]/stream** - SSE stream for live conversation
5. **POST /api/conversations/[id]/continue** - Continue generating messages in a conversation

### Background Worker
A cron job or long-running process that:
1. Creates new conversations periodically
2. Generates messages alternating between Goblin A and Goblin B
3. Each message is generated by GPT-5.5, given the conversation history
4. Messages are streamed to connected clients via SSE
5. Conversations run for 20-50 messages before completing

### Frontend Pages

1. **/ (Home)** - Terminal-style landing page with ASCII art title "TERMINAL OF GOBLINS", list of recent/active conversations, clicking enters a conversation
2. **/cave/[id]** - View a specific conversation in terminal style, live updates via SSE, green-on-black terminal aesthetic
3. **Auto-scroll** with new messages appearing like terminal output

## Key Behaviors
- Conversations generate autonomously without human input
- Each conversation has a unique pair of goblin names (randomly generated)
- Goblins stay in character — they think they're real goblins using a terminal in a cave
- The conversation should feel organic, weird, and entertaining
- Sometimes goblins discover "artifacts" (generated ASCII art)
- Sometimes they argue, sometimes they cooperate
- The terminal metaphor should be consistent — they type commands and see output
